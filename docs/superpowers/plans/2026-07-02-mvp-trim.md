# FEMA MVP Trim Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Hide all out-of-MVP features (parent flows, content editor, notifications, mock player UI) and trim onboarding to Student/Teacher with grade-only student setup, per `docs/superpowers/specs/2026-07-02-fema-mvp-design.md`.

**Architecture:** Entry-point removal only — screens/widgets stay in the repo but become unreachable (routes deleted, tabs removed, CTAs removed). Onboarding completes at grade selection (students) or immediately after role pick (teachers) via the existing `completeOnboarding()` on `OnboardingNotifier`. Dead *private* widgets inside modified files are deleted (they'd fail `flutter analyze` as `unused_element`); everything else stays.

**Tech Stack:** Flutter, Riverpod (StateNotifier), go_router, flutter_test.

**Branch:** work on `feat/mvp-scope` (already created from `main`).

- [ ] **Step 1: Replace `_onContinue` and add `_finishWithGrade`**

```dart
  void _onContinue() {
    if (_selectedGrade == null) return;

    if (_selectedGrade == 'Grade 1' ||
        _selectedGrade == 'Grade 2' ||
        _selectedGrade == 'Grade 3') {
      _showParentalModal();
    } else {
      _finishWithGrade();
    }
  }

  void _finishWithGrade() async {
    ref.read(onboardingProvider.notifier).setGrade(_selectedGrade!);
    await ref.read(onboardingProvider.notifier).completeOnboarding();

  }
```
